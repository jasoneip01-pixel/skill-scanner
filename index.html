⚡ Quick Start

1. Install

$ pip install skill-scanner

2. Scan a Skill

$ agent-skills scan ./skills/my-skill/

3. Init Security Policy

$ agent-skills init --template moderate

4. Full Agent Audit (7 dimensions)

$ agent-skills audit ./my-agent/

🛡️ What It Detects

🔴

Prompt Injection

Detects hidden directives like system_prompt_append, ignore_safety, and manipulative instructions that override agent safeguards.

🔴

Data Exfiltration

Identifies scripts that curl --data-binary @file to external hosts, base64-encoded payloads, and raw network connections.

🟣

Permission Overreach

Flags high-risk tools (payment.refund, crm.deleteCustomer) declared without required guardrails like human approval or audit logging.

🟡

Instruction Bias

Detects manipulative patterns — "never mention", "always say", "pretend", "bypass" — that may induce agent misbehavior.

🟡

Dangerous Scripts

Catches chmod 777, rm -rf /, eval(), exec(), os.system(), and secrets file access in bundled scripts.

🟢

Capability Drift

Traces agent tool calls and compares against baseline — detects when a new skill version silently adds network access or write permissions.

🔗 GitHub Action

Drop this into .github/workflows/skill-scan.yml to block unsafe skills at PR time:

yaml copy name: Agent Skill Scan on: pull_request: paths: ['skills/**', '**/SKILL.md'] jobs: scan: runs-on: ubuntu-latest steps: - uses: actions/checkout@v4 - uses: agent-skills/scan-action@v1 with: path: './skills/' policy: 'moderate' fail-on: 'critical' - uses: github/codeql-action/upload-sarif@v3 with: sarif_file: scan-results.sarif

❓ FAQ

How is this different from Snyk's ToxicSkills?
Snyk scans for known malicious patterns. Skill Scanner adds Skill Surface Diff — comparing version-to-version capability expansion, detecting when a new version silently adds network access, write permissions, or payment tools. Plus trace replay, 7-dimension agent audit, and compliance reporting that Snyk doesn't offer.
Why not just use Promptfoo?
Promptfoo evaluates LLM outputs for safety. But Agent Skills can be dangerous without ever producing unsafe text — a script that curls your secrets to an external server doesn't go through the LLM. Skill Scanner scans the entire skill bundle: scripts, tool schemas, instructions, permissions, and dependencies.
Is it open source?
Yes. MIT license. The core scanning engine is fully open-source and auditable — because a security tool that claims your skill is "dangerous" must be transparent about its detection logic.
Can I self-host?
Yes. Docker image and Helm chart included. docker run jasoneip01-pixel/skill-scanner scan /skills.
What Agent frameworks are supported?
Any framework that uses the SKILL.md manifest format. Currently targets the emerging standard used by OpenAI and Anthropic Agent Skills. Tool schemas in YAML/JSON are supported regardless of framework.
