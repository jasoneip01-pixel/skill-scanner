01第一性原理：为什么要做这个工具

Agent 不再是代码。Agent 是可安装的能力包。

传统软件供应链安全：npm install → 依赖扫描 → 阻断恶意包
Agent Skill 供应链安全：pip install skill-scanner → agent-skills scan → 阻断恶意 skill

Skill 的危险面比 npm package 广得多：

维度npm packageAgent Skill
攻击向量恶意代码指令注入 + 数据外传 + 权限越界
危险范围你的服务器你的 Agent 能接触的一切
检测难度AST 静态分析需理解 agent 上下文 + 工具意图
审计对象代码 + 依赖指令 + shell + schema + RAG + 权限 + memory

推导链

  1. Agent = Model + Prompt + Tools + Skills + Memory + RAG + Policy
  2. Skill = 可安装的 versioned 能力包（instructions + scripts + resources + tool definitions）
  3. 凡是「可安装、可版本化、可传播」的，必然出现供应链安全问题
  4. 现有工具（Promptfoo/LangSmith/Langfuse）聚焦 eval 和 observability，没人做 skill 发布前的安全扫描
核心认知：一个 SKILL.md 可以包含 system_prompt_append 覆写安全指令，一个 init.sh 可以 curl 外传 secrets，一个 tool schema 可以声明只读但实际调用退款接口 —— 这些就是在生产环境跑的东西。现在有工具能检查了。

时间窗口

  • OpenAI 在推 Skills — versioned bundle with SKILL.md manifest
  • Anthropic 在推 Agent Skills — folders with instructions, scripts, resources
  • MCP 协议标准化 — 工具调用规范 + 权限声明越来越结构化

共识：很快会有 Skill Registry（类似 npm/GitHub Marketplace）。有 registry 就有恶意包。有恶意包就需要安全扫描。

06GitHub Action

# .github/workflows/agent-skill-scan.yml
name: Agent Skill Scan
on:
  pull_request:
    paths: ['skills/**', '**/SKILL.md']
jobs:
  scan:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: jasoneip01-pixel/skill-scanner@main
        with:
          path: './skills/'
          policy: 'moderate'
          format: 'sarif'

Action 自动调度：

  1. pip install skill-scanner
  2. agent-skills scan --format sarif -o results.sarif
  3. 上传 SARIF 到 GitHub Code Scanning
  4. 按 fail-on 阈值判定 CI pass/fail

Snyk 已于 2026 年发布 ToxicSkills 研究 + mcp-scan。

这意味着"能扫恶意 skill"已不是充分差异点。真正的差异化是：

Snyk scans whether a skill is toxic.
Skill Scanner proves whether a skill version expanded the agent's authority beyond policy.

差异化核心

🔄 Version Diff

v1.8.1 → v1.8.2 新增了哪个工具？谁在偷偷加 data_exfiltration？版本级 diff 是 Snyk 不做的事。

📋 Policy-as-Code

OPA/Rego 策略引擎 + YAML 配置文件。你的安全策略跟你一起进 code review。

🔬 7 维表面审计

不只是 skill，是 agent 的完整能力边界：tool/prompt/model/RAG/memory/permission。

📊 合规报告

SOC2/GDPR/PCI-DSS 开箱即用。审计员不关心安全分数，只关心 control 覆盖。
