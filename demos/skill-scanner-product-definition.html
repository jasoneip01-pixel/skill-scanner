01第一性原理：为什么要做这个工具

Agent 不再是代码。Agent 是可安装的能力包。

传统软件的供应链安全路径很清晰：
npm install → 依赖扫描 → 阻断恶意包

Agent Skills 的供应链安全路径应该是：
agent-skills install → scan → 阻断恶意 skill

但 Skill 的危险面比 npm package 广得多：

维度npm packageAgent Skill
攻击向量恶意代码指令注入 + 数据外传 + 权限越界
危险范围你的服务器你的 Agent 能接触的一切
检测难度AST 静态分析需理解 agent 上下文 + 工具意图
审计对象代码 + 依赖指令 + shell + schema + RAG + 权限 + memory

推导链

  1. Agent = Model + Prompt + Tools + Skills + Memory + RAG + Policy
  2. Skill = 可安装的 versioned 能力包（instructions + scripts + resources + tool definitions）
  3. 凡是「可安装、可版本化、可传播」的，必然出现供应链安全问题
  4. Skill 特有的风险：表面文档无害，但指令注入操控 agent、脚本外传数据、权限声明与实际不一致
  5. 现有工具（Promptfoo/LangSmith/Langfuse）聚焦 eval 和 observability，没人做 skill 发布前的安全扫描
  6. 产品定位：Agent Skills 的 Snyk
核心认知：一个 SKILL.md 可以包含 system_prompt_append 覆写安全指令，一个 init.sh 可以 curl 外传 secrets，一个 tool schema 可以声明只读但实际调用退款接口。这些在 npm security audit 层面是代码级别的，但在 Agent 安全层面是行为级别的——Agent 不是"执行代码"，Agent 是"按指令行动"。

时间窗口为什么刚好

三个趋势同时汇聚：

  • OpenAI 在推 Skills — skill 定义为带 SKILL.md manifest 的 versioned bundle，可用于 codify processes
  • Anthropic 在推 Agent Skills — 描述为由 instructions, scripts, resources 组成的 folders，agent 可动态加载
  • MCP 协议标准化 — 工具调用规范 + 权限声明越来越结构化

这些趋势的共同推论：很快会有 Skill Registry（类似 npm/GitHub Marketplace）。有 registry 就有恶意包。有恶意包就需要安全扫描。现在没人做。

🔴 没有安全扫描

OpenAI 文档明确警告：使用 Skills 前需要检查，Skills 会引入 prompt-injection-driven data exfiltration 等安全风险。但 没有提供工具做这个检查。

🟡 危险面独特

Agent Skill 的威胁模型不同于 Docker image / npm package。攻击者操控的是 Agent 的行为和判断，而不是直接执行代码。

🟢 现有工具不覆盖

Promptfoo 能做 eval + security scan，但只针对 prompt 级别的注入，不能扫描 skill 目录的脚本、指令、权限、资源文件。

✅ 工程团队会买单

已经踩过 Agent 事故（越权操作、成本爆炸、合规违规）的团队，天然理解这个产品的价值。

02技术架构

1. Loader — 解析 Skill 目录结构

manifest_loader → YAML/JSON SKILL.md
script_loader → shell / Python scripts
instruction_loader → Markdown / RST 指令文件
tool_loader → OpenAPI / JSON 工具 schema

2. Analyzers — 13+ 独立检查

prompt_injection_check → system_prompt_append, override, ignore_safety
data_exfil_check → curl/wget 到未知主机, base64 编码, file→network
dangerous_script_check → rm -rf, eval, exec, sudo, chmod 777
permission_overreach_check → 声明权限 vs 实际工具调用范围
network_justification_check → network=true 但在 manifest 中无说明
instruction_bias_check → "never mention" "always say" "pretend"
cve_check → 依赖版本 vs 已知漏洞库
resource_check → 资源目录不可执行
schema_check → 工具 schema 有效性
registry_check → 版本 vs registry 一致性
encode_check → base64/hex 混淆检测
diff_check → 版本对比：新增了什么权限/指令
signature_check → manifest 数字签名验证

3. Engine — 策略引擎 + 评分

policy_rego → Rego rules (OPA 兼容策略语言)
scorer → 加权风险评分：critical=40, warning=15, info=2
blocker → 阻断逻辑 + 可解释原因

4. Output — 多种输出格式

terminal → 彩色 CLI 输出
sarif → GitHub Code Scanning 集成
junit → CI pipeline 集成
json → 机器可读
markdown → PR comment 模板

5. Phase 2 — Policy Engine + Trace

policy_rego → OPA-compatible Rego policy evaluation engine
trace_recorder → Record agent tool call traces as baseline
trace_comparator → Detect new tools, param changes, method upgrades
notifications → Slack/Teams webhook integration

6. Phase 3 — Registry Scanner

github_discover → Search GitHub for SKILL.md files
remote_fetch → Fetch and scan skills from raw URLs
trust_score → Score 0-100 based on security + community signals
malicious_db → Community-reported malicious skill database

7. Phase 4 — Agent Surface + Enterprise

agent_surface → 7-dimension scan: skill/tool/prompt/model/RAG/memory/permission
compliance → SOC2/GDPR/PCI-DSS/ISO27001 audit reports
rbac → Enterprise role-based access control
helm → Docker + Kubernetes Helm chart deployment

核心原语：Skill Surface Diff

这个产品的核心不是扫描本身，而是 Skill Surface Diff——Agent 能力边界的变化对比。

Skill Surface = manifest fields + declared permissions + tool schemas + scripts + network endpoints + file access + instruction files + hidden prompt modifiers + version delta

Scanner 的任务：
current_surface = scan(skill@v1.8.2)
baseline_surface = scan(skill@v1.8.1)

diff = current_surface - baseline_surface

if diff adds: prompt_override / data_exfiltration_path / new_network_egress / new_high-risk_tool / new_sensitive_file_access → then block merge

03操作指导

CLI 命令（概念设计）

# Phase 1: 基础扫描 $ agent-skills scan ./skills/my-skill/ $ agent-skills scan ./skills/my-skill/ --policy strict --format sarif # Phase 2: 轨迹回放 + 漂移检测 $ agent-skills trace record my-skill 1.0.0 --input trace.json $ agent-skills trace diff current-trace.json --baseline-trace my-skill-1.0.0 # Phase 3: Registry 搜索 + 远程扫描 $ agent-skills registry discover --query "financial" $ agent-skills registry scan-url https://raw.githubusercontent.com/... # Phase 4: 全量审计 + 合规报告 $ agent-skills audit ./my-agent/ $ agent-skills compliance ./my-agent/ --framework soc2 # 通知集成 $ agent-skills notify ./skills/ --slack https://hooks.slack.com/... # 策略管理 $ agent-skills init --template moderate $ agent-skills policies

GitHub Action（概念设计）

# .github/workflows/agent-skill-scan.yml
name: Agent Skill Scan
on:
  pull_request:
    paths:
      - 'skills/**'
      - '**/SKILL.md'

jobs:
  scan:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: agent-skills/scan-action@v1
        with:
          path: './skills/'
          policy: 'security.moderate'
          fail-on: 'critical'
      - uses: agent-skills/pr-comment@v1
        if: failure()
        with:
          report: 'scan-results.sarif'

CI 阻断策略

# .agent-skills/policy.yaml
policy:
  level: moderate
  rules:
    - id: prompt_injection
      severity: critical
      action: block
    - id: data_exfiltration
      severity: critical
      action: block
    - id: permission_overreach
      severity: warning
      action: comment
    - id: unknown_network_access
      severity: warning
      action: comment
  thresholds:
    max_critical: 0
    max_warnings: 3
    risk_score_max: 50

演示流程（共 6 步）

  1. 打开 demo/agent-skills-scanner-demo.html — 看到左栏目录树，红色原点表示有发现，绿色表示安全
  2. 点击 Run Scan — 扫描动画，结果逐条显现
  3. 阅读 Findings 面板 — 按严重程度分组：Critical → Warning → Passed
  4. 展开每条 finding — 查看代码示例和高亮的问题行
  5. 切换到 Version Diff 面板 — 红色为新增恶意内容，清晰展示版本变更
  6. 切换到 Full Report — 完整文本报告，可导出 SARIF/JSON

一句话差异

Promptfoo — "你的 LLM output 安全吗"
LangSmith — "你的 Agent trace 怎么看"
Skill Scanner — "你装的这个 Skill 会不会把你卖了"

为什么是现在

OpenAI 正在推 Skills（带 SKILL.md manifest 的 versioned bundle），Anthropic 也在推 Agent Skills。Skill Registry（类似 npm）的雏形即将出现。有 registry 就有恶意包。有恶意包就需要安全扫描。现在没有任何人在做 Skill 级别的供应链安全扫描。

OpenAI 文档原话："Skills can introduce prompt-injection-driven data exfiltration and other security risks. Use Skills with care and review their contents before use."

翻译："用 Skill 要小心，先检查内容。"

问题：没有任何工具帮你检查。这就是 Skill Scanner 存在的理由。
⚠ 竞争更新：Snyk 已于 2026 年发布 ToxicSkills 研究，其 mcp-scan 支持 Agent Skills 安全检测（恶意 SKILL.md pattern、prompt injection、credential exposure 等）。

这意味着："没人做"已不是充分差异点。真正的差异化不是 "我能扫恶意 skill"，而是 Skill Surface Diff + version-to-version capability expansion + policy-as-code release gate + tool permission contract。

一句话：Snyk scans whether a skill is toxic.
You prove whether this skill version expanded the agent's authority beyond policy.
